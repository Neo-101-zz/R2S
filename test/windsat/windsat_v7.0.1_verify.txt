This verification file is for Version 7 Windsat data files (released November 2011)

PLEASE read any README.txt files in this and the previous directories
prior to using this file.

The following data can be used to determine if your adapted processing
routine is correctly reading the RSS WindSat binary data file.


To accurately compare these data, you need to download the respective file to check:

Time              Directory                        	File
daily passes      windsat/bmaps_v07/y2006/m04/       	windsat_20060409v7.gz
3-day mean        windsat/bmaps_v07/y2006/m04/      	windsat_20060409v7_d3d.gz
weekly mean       windsat/bmaps_v07/weeks/           	windsat_20060415v7.gz
monthly mean      windsat/bmaps_v07/y2006/m04/       	windsat_200604v7.gz

Each of these files contains WindSat data for:
the day April 9th, 2006 (daily)
the days April 7, 8 and 9th, 2006 (3-day mean)
April 9th (Sunday) to April 15th (Saturday), 2006 (weekly mean) 
or the month of April 2006 (monthly mean) 

Once you have obtained the correct file, read it with your program and 
write out the data for the following subsets:


Fortran, Matlab and other 1-based languages:
Daily:  (170:175,274:278,1:9,1) from the full array sized (1440,720,9,2)
Mean:   (170:175,274:278,1:8)   from the full array sized (1440,720,8)

IDL, and other 0-based languages:
Daily:  (169:174,273:277,0:8,0) from the full array sized (1440,720,9,2)
Mean:   (169:174,273:277,0:7)   from the full array sized (1440,720,8)


mingmt, SST, WSPD-LF, WSPD-MF,VAPOR, CLOUD, RAIN, WSPD-AW, and WDIR values for this longitude, latitude subset: 
	lon 170:175  (6 cells)
	lat 274:278  (5 cells)
	iasc = 1     (morning (descending) passes)


The binary file contains numbers 0 to 255
0 - 250 are valid geophysical data.
The remaining 5 numbers are coded: 
251 = areas in which no data was processed
252 = sea ice > 0%
253 = bad data such as along coastline regions
254 = regions of no observations
255 = land


These data are for a small section in the 
Mozambique Channel just west of Madagascar and 
is shown in the windsat_V7_verify.png image.
The pixels listed above are shown in the red rectangle.
The first set of data in this list is located in 
the bottom left of the red rectangle.

In both the image and the data listed below, you will 
notice that there are cells for which there are no wind speeds 
but there are a wind directions.  This is because of the 
different land masks and effective resolutions of the 
channels used to make each product.  Please be sure to
read the description provided on the web site.
www.remss.com/windsat/windsat_data_description.html


LL 42.375,-21.625  to  UR 43.625,-20.625
The array position is related to latitude and longitude by:
  XLAT= 0.25*ILAT - 90.125
  XLON= 0.25*ILON -  0.125


The time is minute of day GMT (mingmt) and has 
valid values between 0 and 1440 (0 - 250 binary
value * 6. scale factor).  A value greater than 
1440 is representative of no data.  

All read routines return -999.0 for all missing data.
feel free to adapt the routines as needed.


If you have any problems with any of the data reading routines contact RSS support:
http://www.remss.com/support
